# 爬蟲技術與網路資料抓取

## JSON

JSON 的全名是 JavaScript Object Notation（JavaScript 物件表示法）。雖然名稱中帶有 JavaScript，但它是一種「語言無關」的純文字格式。在現代網頁開發中，JSON 扮演著「國際通用語」的角色。當你打開一個手機 App（如天氣預報或 IG）時，後端伺服器並不是傳送一張圖片或一個漂亮的網頁給手機，而是傳送一串 JSON 格式的文字，手機端再將這串文字解析並顯示在螢幕上。

早期的網頁中，資料交換常使用 XML，近年來JSON因為以下原因成為主流：

- 輕量化：比起 XML，JSON 的冗餘標籤極少，節省頻寬。
- 原生支持：JavaScript 可以直接將 JSON 轉換為物件（Object），不需要複雜的解析過程。
- 易讀性：結構清晰，人類與機器都能輕鬆閱讀。

### JSON 的核心結構：鍵值對與列表
JSON 的結構非常單純，主要由以下兩大支柱構成：

1. 物件 (Object) / 鍵值對 (Key-Value Pairs)：
在 JSON 中使用大括號 {} 包圍。每一個資料都有一個「名字」（Key）和對應的「內容」（Value）。
    - 規則： Key 必須是字串且用雙引號 "" 包圍；Value 可以是字串、數字、布林值、null，或是另一個物件或列表。
    - 例子： {"name": "ytlAi Lab", "year": 2026}

2. 陣列 (Array) / 列表 (Lists)：
使用中括號 [] 包圍，用來存放一系列有序的資料。
    - 例子： ["Python", "AI", "Crawler"]

### JSON 與 Python 字典的相似性

對於 Python 學習者來說，JSON 的格式與 Python 的 字典 (Dictionary) 和 串列 (List) 高度重合。
- JSON 的 {} 對應 Python 的 dict。
- JSON 的 [] 對應 Python 的 list。
- 關鍵差異：
  1. 引號： JSON 規定必須使用雙引號 "，Python 則單雙引號皆可。
  2. 布林值： JSON 為 true/false（小寫），Python 為 True/False（大寫）。
  3. 空值： JSON 為 null，Python 為 None。
 
### 處理 JSON 字串 (反序列化)
當我們從網路爬取資料時，拿到的通常是「字串」格式。我們必須先使用 json.loads() (load string) 將其轉換為 Python 字典。
```
import json

# 模擬從網路上抓取下來的文字資料 (JSON String)
json_string = """
{
    "course": "Python 爬蟲技術",
    "week": 5,
    "is_active": true,
    "teacher": "Jessica"
}
"""

# 將文字轉換為 Python 字典
data = json.loads(json_string)

print(f"資料型態: {type(data)}")
print(f"課程老師: {data['teacher']}")
```

### 挑戰巢狀結構 (Nested Structures)
現實世界的爬蟲資料通常是「大包裝小包」。例如一個購物網站的資料，可能包含：商店資訊 -> 商品列表 -> 每個商品的規格。這就是所謂的巢狀結構。

實作範例：提取實驗室成員資訊 假設我們抓取到一段包含 ytlAi Lab 成員與設備的資料：
```
import json

complex_json = """
{
    "lab_name": "ytlAi Lab",
    "location": "Taipei",
    "inventory": {
        "hardware": [
            {"type": "iPad", "count": 64, "status": "reserved"},
            {"type": "MacBook Pro", "count": 5, "status": "available"}
        ],
        "software": ["Python", "Node-RED", "Google Colab"]
    },
    "manager": {
        "name": "Jessica",
        "certified": ["Level 1", "Level 2", "Gemini Educator"]
    }
}
"""

# 解析資料
lab_data = json.loads(complex_json)

# 練習提取：
# 1. 取得 lab 名稱
print(f"實驗室名稱: {lab_data['lab_name']}")

# 2. 取得第二種硬體的名稱 (MacBook Pro)
# 路徑: inventory -> hardware -> 第2個元素 -> type
second_hw = lab_data['inventory']['hardware'][1]['type']
print(f"硬體資產: {second_hw}")

# 3. 遍歷並印出所有軟體
print("實驗室使用軟體:")
for soft in lab_data['inventory']['software']:
    print(f"- {soft}")
```

### JSONPlaceholder
JSONPlaceholder 是一個免費的線上假資料 (Mock API) 服務，提供開發者測試、練習 Ajax、API 串接或製作原型 (Prototype) 而設計。

核心功能與特點：
* 免註冊免費使用： 無需 API 金鑰，直接即可使用。
* 豐富的假資料： 提供包含 Posts (文章)、Comments (留言)、Albums (相簿)、Photos (照片)、Todos (待辦事項)、Users (使用者) 等常見資料。
* 真實的 CRUD 操作： 例如使用 POST 建立資料，看起來會成功，但實際上不會真正將資料存入伺服器，非常適合前端測試流程。
* 支援 HTTPS： 確保在安全環境下進行數據請求。

取得資料 (GET)：獲取貼文內容

這是最常見的操作，模擬從伺服器抓取清單或特定資訊。

```
import requests
import json

# 目標 URL：取得所有貼文中的第一篇
url = "https://jsonplaceholder.typicode.com/posts/1"

# 發送 GET 請求
response = requests.get(url)

# 檢查狀態碼 (200 代表成功)
if response.status_code == 200:
    # 將回傳的 JSON 轉換為 Python 字典
    post_data = response.json()

    print("--- 成功取得貼文 ---")
    print(f"標題: {post_data['title']}")
    print(f"內文: {post_data['body']}")
else:
    print(f"請求失敗，錯誤代碼：{response.status_code}")
```

傳送資料 (POST)：模擬新增貼文

當你需要「上傳」資料到伺服器時會用到 POST。雖然 JSONPlaceholder 不會真的把資料存入資料庫，但它會模擬回傳一個「新增成功」的狀態與 ID。
```
# 目標 URL
url = "https://jsonplaceholder.typicode.com/posts"

# 準備要上傳的資料 (Python 字典格式)
new_post = {
    "title": "ytlAi Lab 測試貼文",
    "body": "這是一篇透過 Python Requests 發送的 JSON 測試內容。",
    "userId": 1
}

# 發送 POST 請求，並指定 json 參數
response = requests.post(url, json=new_post)

if response.status_code == 201: # 201 代表 Created (成功創建)
    print("--- 資料傳送成功 ---")
    print(f"伺服器回傳內容: {response.json()}")
```

## 爬蟲實作

### requests + json

傳統的爬蟲（如 BeautifulSoup）通常是在處理 HTML 標籤。這就像是在一張印滿廣告、側邊欄和裝飾圖案的報紙中，用剪刀剪下你需要的報導。這種方式不但辛苦，且一旦網頁版面稍微變動（例如改個顏色或移動位置），爬蟲程式就可能失效。

然而，現代網頁（尤其是顯示即時資訊的網站，如股市、天氣、地圖）為了效能，通常會採用「前後端分離」的架構。網頁載入時，瀏覽器會先下載畫面的「骨架」，接著再向後端伺服器請求一份純資料（通常是 JSON 格式）來填入內容。這份純資料的連結，就是我們所說的「隱形 API」。直接抓取這個 API，就像是直接向報社訂閱電子報，跳過了所有無用的廣告裝飾，效率高且結構穩定。

瀏覽器開發者工具 (F12)
- 如何進入： 在網頁按下 F12（或右鍵點擊「檢查」），切換到 Network (網路) 標籤。
- 過濾資料： 點選 Fetch/XHR 子分頁，這會過濾掉圖片、樣式檔，只顯示資料交換的請求。
- 尋找資料： 重新整理網頁，觀察出現的項目，點擊後查看 Preview (預覽)。如果看到整齊的 {...} 結構，那就是我們要找的資料源。

在 Python 中，requests 是處理 HTTP 請求的業界標準。

requests.get(url)： 模擬瀏覽器發出請求。伺服器會回傳一個 Response 物件。

.json() 方法： 這是最神奇的一步。如果伺服器回傳的是 JSON 格式，這個方法會自動將那一長串「字串」轉換為 Python 的「字典 (dict)」或「列表 (list)」，讓我們可以直接進行邏輯運算。

YouBike2.0臺北市公共自行車即時資訊介接網址為

https://tcgbusfs.blob.core.windows.net/dotapp/youbike/v2/youbike_immediate.json

```
import requests

# 這是台北市政府公開的 YouBike 2.0 原始 JSON 資料源
url = "https://tcgbusfs.blob.core.windows.net/dotapp/youbike/v2/youbike_immediate.json"

try:
    # 發送請求
    response = requests.get(url)

    # 解析 JSON
    data = response.json()

    print("連線成功！")
    print("目前台北市共有", len(data), "個 YouBike 站點")
    print("---------------------------------------")

    # 抓取前 5 個站點的名稱與可借車輛
    for station in data[:5]:
        name = station['sna']     # 站名
        bikes = station['sbi']    # 可借車數
        total = station['tot']    # 總車格

        print("站點:", name)
        print("剩餘車輛:", bikes, "/", total)
        print("-" * 20)

except Exception as e:
    print("連線失敗，錯誤原因:", e)
```

#### 實作網頁：
[中央氣象署 - 每 3 小時天氣預報](https://www.cwa.gov.tw/V8/C/W/County/County.html?CID=63)

操作指引：

開啟開發者工具 (F12)：切換到 Network (網路) 標籤。

有時候檔案被歸類在 JS 標籤下，而非 XHR。

1. 在開發者工具中，點擊 「JS」 標籤。
2. 在搜尋框輸入：F-C0032-001 (這是常見的預報代碼) 或 County。
3. 尋找一個名為 W50_Data.js 的檔案，這通常是目前網頁存放各縣市預報的資料源。


解碼

https://www.cwa.gov.tw/Data/js/fcst/W50_Data.js
```
import requests
import ast
import re
import json

# 1. 設定目標 URL 與 Headers
url = "https://www.cwa.gov.tw/Data/js/fcst/W50_Data.js"
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36",
    "Referer": "https://www.cwa.gov.tw/"
}

# 2. 抓取資料並強制轉碼
response = requests.get(url, headers=headers)
response.encoding = 'utf-8'
raw_text = response.text

# 3. 擷取 JSON 資料區塊 (過濾 var ... = 與結尾的 ; App.init();)
# 我們尋找第一個 '{' 到最後一個 '}'
json_match = re.search(r'(\{.*\})', raw_text, re.DOTALL)

if json_match:
    content = json_match.group(1)

    # 再次清理：確保只抓到大括號結尾，排除可能存在的 App.init()
    last_brace = content.rfind('}') + 1
    clean_json_str = content[:last_brace]

    try:
        # 4. 將 JS 物件字串轉換為 Python 字典
        # 使用 ast.literal_eval 處理非標準 JSON (Key 未加雙引號)
        all_data = ast.literal_eval(clean_json_str)

        # 5. 遍歷所有縣市並印出 (不使用格式化語法)
        # 取得所有縣市代碼 (如 '63', '64'...)
        counties = all_data.keys()

        for cid in counties:
            county_data = all_data[cid]

            print("--- 縣市代碼 ---")
            print(cid)

            # 檢查是否有名稱資料
            if 'Name' in county_data:
                print("中文名稱:")
                print(county_data['Name']['C'])
                print("英文名稱:")
                print(county_data['Name']['E'])

            # 檢查是否有預報資料
            if 'Forecast' in county_data:
                print("預報時段數量:")
                print(len(county_data['Forecast']))

                # 輸出該縣市第一個預報時段作為範例
                first = county_data['Forecast'][0]
                print("最新時段天氣:")
                print(first['Time'])
                print("天氣現象:")
                print(first['Wx']['C'])
                print("氣溫範圍:")
                print(first['MinT']['C'])
                print("-")
                print(first['MaxT']['C'])
                print("度")

            print("==============================")

    except Exception as e:
        print("解析失敗，錯誤原因:")
        print(e)
        # 如果失敗，印出末尾內容檢查格式
        print("資料末尾片段:")
        print(clean_json_str[-100:])
else:
    print("無法從網頁內容中找到有效的資料大括號區塊")
```

import requests
import re
import ast

url = "https://www.cwa.gov.tw/Data/js/fcst/W50_Data.js"
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36",
    "Referer": "https://www.cwa.gov.tw/"
}

response = requests.get(url, headers=headers)
response.encoding = 'utf-8'
raw_text = response.text

# 擷取 JS 物件
json_match = re.search(r'(\{.*\})', raw_text, re.DOTALL)

if json_match:
    content_str = json_match.group(1)
    last_brace = content_str.rfind('}') + 1
    clean_data_str = content_str[:last_brace]

    try:
        full_data = ast.literal_eval(clean_data_str)
        taipei_content = full_data['63']['Content']

        print("查詢區域: 臺北市")
        print("資料更新時間:")
        print(full_data['63']['DataTime'])
        print("---------------------------")

        # 處理純文字列表
        for text_item in taipei_content:
            # text_item 範例: "今日白天（27日）天氣晴時多雲，降雨機率0%，氣溫18至32度..."
            print("原始文字預報:")
            print(text_item)

            # 使用正則表達式抓取氣溫數字 (尋找 "氣溫" 後面的數字)
            # \d+ 代表一個以上的數字
            temps = re.findall(r"氣溫(\d+)至(\d+)度", text_item)

            if temps:
                # temps 會是一個 tuple 列表，例如 [('18', '32')]
                low_temp = temps[0][0]
                high_temp = temps[0][1]

                print("解析後的最低溫:")
                print(low_temp)
                print("解析後的最高溫:")
                print(high_temp)
                print("度")

            print("---")

    except Exception as e:
        print("最終解析失敗:")
        print(e)
else:
    print("無法定位資料區塊")

### pandas 結構化網頁表格的抓取

pd.read_html()
如果要抓取的資料在網頁上是以「表格（Table）」形式呈現（例如匯率表、運動賽事排名、維基百科的統計數據），可使用 pandas 的 read_html 方法。

這個函數的強大之處在於，它會自動掃描網頁原始碼中所有的 table 標籤，並試圖將它們全部轉換為 Python 中最適合做數據分析的格式——DataFrame 完全不需要處理 HTML 解析、標籤定位，只要一行程式碼，資料就變成了像 Excel 一樣的表格。

使用限制與優勢
- read_html 極度方便，但並非萬能。
-優勢： 速度極快、程式碼精簡、自動處理表頭（Header）與資料型態（例如將數字字串轉為 float）。
- 限制： 只能抓取網頁原始碼中「寫死」的 table。如果網頁是透過 JavaScript 動態生成的表格（即在原始碼找不到 tr 標籤），此方法就抓不到資料。此外，對於排版複雜的網頁，它可能會抓到許多用來排版的「垃圾表格」，需要額外進行資料清洗。

```
import pandas as pd

url = "https://rate.bot.com.tw/xrt?Lang=zh-TW"

# 抓取表格
dfs = pd.read_html(url)
df = dfs[0]

# 直接印出原始欄位數量與前幾列確認結構
print("原始欄位數量:")
print(len(df.columns))

# 處理多層表頭：台銀的表格通常有 MultiIndex 或重複的欄位
# 我們先強制將欄位平坦化，只取我們需要的索引位置
# 通常前 5 欄是：幣別、現金買(本行)、現金賣(本行)、即期買(本行)、即期賣(本行)
new_df = df.iloc[:, [0, 1, 2, 3, 4]].copy()

# 重新命名這 5 欄
new_df.columns = ['幣別', '現金-買入', '現金-賣出', '即期-買入', '即期-賣出']

# 清理幣別文字 (移除多餘換行與重複字串)
new_df['幣別'] = new_df['幣別'].str.extract(r'([A-Z]+)')

# 輸出結果
print("處理後的匯率資料:")
print(new_df)
```

### 使用API爬取
#### PokeAPI
PokeAPI (https://pokeapi.co/) 是一個為開發者提供的免費、開源 RESTful API 介面，內容涵蓋了《寶可夢》系列遊戲中幾乎所有的數據。其中整理了極其詳盡的資料庫，包括但不限於：

- 寶可夢資料： 名字、屬性、身高、體重、基礎能力值（HP、攻擊等）、進化鏈。
- 技能與物品： 技能威力、命中率、效果說明、各種道具的功能。
- 遊戲機制： 特性（Abilities）、性格（Natures）、蛋組、屬性相剋表。
- 多媒體資源： 提供各代遊戲的圖片（Sprites）網址連結。

取得特定寶可夢的基本資料

這個範例會抓取「皮卡丘」的名字、身高、體重以及所屬屬性。
```
import requests

# 設定 API 網址 (以皮卡丘為例)
url = "https://pokeapi.co/api/v2/pokemon/pikachu"

# 發送請求

#### OpenWeatherMap

OpenWeatherMap API (https://openweathermap.org/api) 是一個全球性的氣象數據服務平台，廣泛應用於開發天氣預報 App、農業監測系統或物聯網 (IoT) 專案。

網頁功能與特點說明
1. 多樣化的 API 服務：
    - Current Weather Data： 獲取全球超過 20 萬個城市的即時天氣。
    - 5 Day / 3 Hour Forecast： 獲取每 3 小時更新一次的五天天氣預報。
    - Air Pollution API： 提供空氣品質指數 (AQI) 及污染物數據（如 PM2.5, NO2）。
    - Geocoding API： 用於將城市名稱轉換為經緯度座標。

2. API Key 機制：
    - 與 PokeAPI 不同，OpenWeatherMap 必須註冊帳號並取得 API Key 才能呼叫資料。
    - 免費版 (Free Plan) 每分鐘限制 60 次請求，每月 100 萬次，這對於教學與個人開發已非常足夠。

3. 資料結構：
    - 回傳格式主要為 JSON（也支援 XML 或 HTML）。
    - 包含豐富的氣象指標：溫度、體感溫度、濕度、氣壓、風速、雲量及日出日落時間。

取得特定城市目前的即時天氣

以「台北 (Taipei)」為例

```
import requests

# 請確保這串 Key 是正確的

city = "Taipei"

response = requests.get(url)

# 先檢查回傳狀態碼
if response.status_code == 200:
    # 成功取得資料才進行提取

    weather_desc = data["weather"][0]["description"]

else:
    # 如果失敗，印出錯誤訊息

    print("錯誤訊息內容:", data)

    "66": "臺中市",
    "67": "臺南市",
    "64": "高雄市"

    last_brace = content_str.rfind('}') + 1

    try:
        all_data = ast.literal_eval(clean_data_str)

import pandas as pd
import requests
import io
from urllib.parse import quote

# 1. 抓取資料
base_url = "https://zh.wikipedia.org/wiki/"
path = "各国国内生产总值列表_(国际汇率)"

headers = {"User-Agent": "Mozilla/5.0"}
response = requests.get(url, headers=headers)
dfs = pd.read_html(io.StringIO(response.text))

# 2. 自動尋找正確的數據表格
# 我們遍歷所有表格，尋找長度大於 150 (全球國家數接近 200) 且包含數據的表格

    df = dfs[2]

# 3. 強制簡化欄位名稱 (避免 MultiIndex 造成困擾)
df.columns = [f"col_{i}" for i in range(len(df.columns))]

# 4. 根據維基百科該頁面結構：
# col_0: 國家/地區
# col_2: IMF 估計值 (注意：有些版本 col_1 是地區，col_2 才是數值)
final_df = df[["col_0", "col_2"]].copy()
