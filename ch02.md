# 資料結構與邏輯控制進階

## 資料結構：Dictionary (字典)
在 Python 的 List (清單)，它像是一列火車，資料按順序排列。但在處理現實世界的複雜資料（如：會員資料、股票行情、天氣預報）時，順序往往不是重點，**「對應關係」**才是關鍵。這就是 Dictionary (字典) 出現的原因。

字典是由 Key (鍵) 與 Value (值) 組成的成對結構。想像你在翻閱實體字典：你想查「Apple (Key)」，字典就會告訴你「蘋果 (Value)」。這種存取方式不依賴索引編號（0, 1, 2...），而是直接搜尋關鍵字，因此在處理數萬筆資料時，效率極高。

在處理 Web API（如財經數據、氣象資料）時，資料幾乎都是以 JSON 格式回傳，而在 Python 中，JSON 對應的就是 Dictionary。

### 字典的核心觀念
Key-Value Pair (鍵值對)：想像成一本字典，透過「單字 (Key)」查到「定義 (Value)」。
特性：
- Key 必須是唯一的（通常是字串）。
- 存取速度極快，不受資料量大小影響。
- 無序性（在 Python 3.7+ 後維持插入順序，但邏輯上仍透過 Key 存取而非索引）。

### 範例1 字典的建立
如何讀取字典、新增、修改、刪除字典資料
```
# 範例1 字典的建立
# 建立字典
student_info = {
    "name": "ytl",
    "id": "A1234567",
    "courses": ["Python", "AI", "Data Science"],
    "is_active": True
}

# 1. 讀取資料 (兩種方法)
print(student_info["name"])        # 方法一：直接讀取（Key不存在會報錯）
print(student_info.get("age", 25)) # 方法二：使用 .get()，若Key不存在可回傳預設值

# 2. 新增與修改
student_info["age"] = 30           # 新增
student_info["is_active"] = False  # 修改

# 3. 刪除資料
del student_info["id"]
```

## 邏輯控制實務：巢狀迴圈與條件判斷
巢狀結構 (Nested Structure) 是指在一個控制流程中包含另一個控制流程。在 Python 實務中，最常見的就是「迴圈內包含判斷式 (If in Loop)」以及「迴圈內包含迴圈 (Loop in Loop)」。

當處理二維資料（如試算表中的行列）或複雜清單（如裝著字典的清單）時，巢狀迴圈是必備工具。外層迴圈通常代表「大單位」（例如：每一列、每一個使用者），內層迴圈則處理「小細節」（例如：每一格、使用者的每一項屬性）。

### 範例1 迴圈內的基礎判斷 (篩選)
從一組數字中找出符合條件的目標。。
```
#範例1 迴圈內的基礎判斷 (篩選)

numbers = [12, 5, 8, 19, 3, 22]

for n in numbers:
    if n > 10:
        print("大於 10 的數字:", n)
```

### 範例2 巢狀迴圈 (九九乘法表)
使用兩個迴圈來產生組合資料。
```
# 範例2 巢狀迴圈 (九九乘法表)
# 只印出 2 和 3 的乘法
for i in [2, 9]:
    for j in [1, 9, 1]:
        print(i, "乘以", j, "等於", i * j)
```

## 錯誤處理 (Try-Except)
在撰寫程式時，程式有時會因為一個小錯誤而完全停止執行。這種錯誤可能是因為使用者輸入了非數字的文字、讀取了不存在的檔案，或是網路爬蟲抓取資料時網頁突然斷線。如果沒有處理這些「例外情況 (Exceptions)」，程式就會發生「崩潰 (Crash)」。

Try-Except 結構是 Python 提供的防護機制。它的邏輯是：嘗試 (Try) 執行一段程式碼，如果發生了指定的錯誤，就執行錯誤處理 (Except) 的備案，而不是直接讓程式關閉。這在開發「強韌」的程式（例如需要跑數小時的自動化交易系統或爬蟲）時至關重要。

請理解：並非所有錯誤都是「寫錯程式」，很多時候是「資料不符合預期」。學習捕捉特定的錯誤類型（如 ValueError, KeyError），能讓我們針對不同問題給予精確的提示。此外，finally 區塊則確保了不論成功或失敗，某些動作（如關閉檔案或資料庫連線）都一定會執行。

核心語法結構

- try：放置可能產生錯誤的程式碼。
- except：當錯誤發生時，執行的應變措施。
- else：如果 try 區塊順利執行完畢（無錯誤），則執行此處。
- finally：無論是否發生錯誤，最後都一定會執行的清理工作（如關閉檔案）。

### 範例1 基本的輸入防錯
處理最常見的資料型態轉換錯誤。
```
# 範例1 基本的輸入防錯
user_input = "abc" # 模擬使用者亂輸入

try:
    number = int(user_input)
    print("成功轉換為數字:", number)
except:
    print("錯誤：您輸入的不是數字，請重新輸入。")
```

### 範例2 捕捉特定的錯誤類型
明確區分不同的錯誤原因，能給予更好的使用者回饋。
```
# 範例2 捕捉特定的錯誤類型
try:
    # 這裡可能發生兩種錯誤
    num = 10 / 0 # 除以零錯誤
except ZeroDivisionError:
    print("錯誤：數學運算中不能除以零。")
except ValueError:
    print("錯誤：數值格式不正確。")
```

### 範例3 處理字典中缺失的鍵 (KeyError)
在處理 API 資料時，某些欄位可能偶爾缺失，這能避免程式中斷。
```
# 範例3 處理字典中缺失的鍵 (KeyError)
stock = {"name": "台積電", "price": 600}

try:
    # 嘗試讀取一個不存在的鍵
    print("股利資訊:", stock["dividend"])
except KeyError:
    print("錯誤：資料中找不到股利資訊。")
```

### 範例4 結合迴圈與錯誤處理 (爬蟲資料)
這是實務中常發生的情境：當處理大量清單資料時，跳過壞掉的資料，繼續處理下一筆。
```
# 範例4 結合迴圈與錯誤處理 (爬蟲資料)
data_list = ["100", "200", "無資料", "400"]

for d in data_list:
    try:
        price = int(d)
        print("有效價格:", price)
    except:
        print("跳過無效資料:", d)
        continue # 繼續處理下一個
```

### 範例5 完整結構 (Try-Except-Finally)

```
# 範例5 完整結構 (Try-Except-Finally)
try:
    print("正在嘗試執行運算...")
    result = 10 / 2
except Exception as e:
    print("發生錯誤:", e)
else:
    print("運算成功，結果為:", result)
finally:
    print("無論如何，這行都會被執行 (常用於關閉檔案或連線)。")
```

### 重點提醒
不要濫用萬用 Except，盡量使用 except ValueError 而非只寫 except，這樣才不會意外隱藏掉真正的程式語法錯誤。
