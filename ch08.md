# Google Apps Script (GAS) 開發

## GAS 開發環境與試算表批次讀寫
### 什麼是 Google Apps Script (GAS)
Google Apps Script 是一款基於 JavaScript (ECMAScript V8 引擎) 的雲端腳本平台。它的核心在於免伺服器維護 (Serverless) 以及與 Google Workspace 服務的無縫整合。對於量化投資者、財務分析師或數據工程師而言，GAS 是一個極佳的輕量級自動化工具，能讓你直接控制 Google 試算表（Spreadsheet）、雲端硬碟（Drive），甚至能透過外部 API 抓取即時盤後股市資料。
由於它是完全免費且運行在 Google 雲端伺服器上，Google 為了防止資源被濫用，設定了嚴格的配額限制 (Quotas & Limitations)：

- 單次執行時間限制： 一般帳號每次執行上限為 6 分鐘（Google Workspace 商業版帳號為 90 分鐘）。
- 每日觸發器執行時間： 每天總觸發執行時間不得超過 1 小時（商業版為 6 小時）。
- URL Fetch 呼叫次數： 每天透過 UrlFetchApp 抓取外部 API（如證交所、財報狗等）的次數有限。

#### GAS 編輯器介面與專案結構
要啟動 GAS 開發環境，最直接的方法是建立一個 Google 試算表，點選上方選單的 「擴充功能」->「Apps Script」。此時會打開一個獨立的雲端開發環境（IDE），介面主要由以下幾個區塊組成：
1. 左側導覽列 (Navigation Bar)：
    - 編輯器 (Editor)： 撰寫 .gs 程式碼與 HTML 檔案的地方。
    - 觸發器 (Triggers)： 設定自動化排程（例如：每天台股收盤後 14:30 自動執行抓取與計算）。
    - 執行項目 (Executions)： 查看歷史執行紀錄、成功與否、消耗時間以及 Logger.log() 的詳細日誌。
    - 專案設定 (Project Settings)： 設定時區、顯示 appsscript.json 資訊清單等。

2. 檔案清單 (Files)： 預設會建立一個 程式碼.gs。你可以自由新增多個 .gs 檔案來分類你的程式邏輯（例如：FetchData.gs, Calculate.gs, UI.gs），執行時 GAS 會將所有 .gs 檔案視為同一個全域範疇 (Global Scope)。
3. 程式碼主編輯器： 支援自動補完（Autocomplete）、語法高亮與錯誤提示。
4. 頂部控制列： 包含「儲存專案（Ctrl+S / Cmd+S）」、「執行 Selected Function」、「偵錯 (Debug)」以及選擇目前要運行的函式下拉選單。

#### 偵錯工具與現代 JavaScript (V8) 特性
在 GAS 中，你不需要 console.log() 到瀏覽器的開發者工具。GAS 提供了專屬的日誌系統：
- Logger.log(value)： 最經典的偵錯手段，適合快速列印變數內容。執行完畢後會在編輯器下方或左側「執行項目」中顯示。
- console.log(value) / console.error(value)： 與 Google Cloud Logging 整合，適合複雜專案，能分級紀錄錯誤資訊。

重要觀念：現代 V8 引擎</br>
過去的 GAS 採用老舊的 Rhino 引擎，不支援許多現代 JS 語法。現在 GAS 預設啟動 V8 引擎，這意味著你可以全面使用以下高效語法，這對陣列批次處理極為重要：

- let 與 const（取代傳統 var，避免變數污染）。
- 箭頭函式 (Arrow Functions)： data.filter(row => row[0] > 100)。
- 解構賦值 (Destructuring Assignment)： const [stockId, price, volume] = row;。
- 樣板字串 (Template Literals)： `股票代號: ${id}, 漲幅: ${change}%`。

### 試算表批次讀寫
在寫 GAS 時，最常見的直覺寫法通常是這樣：用一個 for 迴圈，第 1 行讀取股價、第 2 行計算、第 3 行寫回試算表，接著換下一行。

這種寫法在資料量只有 5 行時沒感覺，但當資料量放大到 200 檔個股時，程式就會變得極度緩慢。
- API 呼叫的沉重成本： 每次你呼叫 sheet.getRange(row, col).getValue() 或 setValue() 時，GAS 伺服器都必須跨網路發送一次請求給 Google 試算表伺服器。這涉及到了網路傳輸、權限驗證以及試算表本身的重新渲染。
- 逐行讀寫的災難： 假設有 200 檔個股，讀取加寫入一共呼叫 400 次 API。每次 API 交互耗時 0.2 秒，光是通訊時間就會吃掉 80 秒，極易遭遇 6 分鐘超時崩潰。
- 批次處理的救贖： 將所有資料「一次性」讀入記憶體變成一個 JavaScript 二維陣列（2D Array），在記憶體中用極速的 CPU 運算完畢後，再「一次性」寫回試算表。這將 API 呼叫次數壓縮到了僅僅 2 次（1 次讀、1 次寫），200 筆資料的處理時間會從 80 秒縮短到不到 1 秒！

#### 二維陣列 (2D Array) ── 記憶體中的試算表
在呼叫批次讀取 sheet.getRange().getValues() 後，你會得到一個二維陣列。
```
// 試算表外觀：
// A列(股票) | B列(現價)
// TSMC     | 600
// Foxconn  | 100

// 讀入記憶體後的二維陣列結構：
[
  ["TSMC", 600],    // 索引 0 (代表試算表第2行，假設第1行是標題)
  ["Foxconn", 100]  // 索引 1
]
```
- 外層陣列 代表「列 (Rows)」。
- 內層陣列 代表「欄 (Columns)」。
- 注意索引偏移 (Index Offset)： 試算表的行列是從 1 開始計數（如 A1 儲存格是 row=1, col=1）；而 JavaScript 陣列的索引是從 0 開始。因此，陣列中的 data[0][0] 對應的是你選取範圍中左上角的第一個儲存格。

#### 批次處理的三步驟
1. [步驟 1: 批次讀取] sheet.getValues() -> 載入二維陣列
2. [步驟 2: 記憶體計算] 透過 for 迴圈或 map() 處理陣列，產生「結果二維陣列」
3. [步驟 3: 批次寫入] sheet.setValues(結果二維陣列) -> 一次性塞回試算表

在寫入時，有一個高頻出錯點：寫入的目標 Range 大小（列數與欄數），必須與結果二維陣列的尺寸完全一致。如果你的陣列是 200 列 x 2 欄，但你指定的 getRange() 只有 200 列 x 1 欄，程式就會毫不留情地拋出 Incorrect range width 錯誤。

#### 程式範例
> 範例一：基礎批次讀寫 ── 計算股市漲跌幅與量能異常

讀取 A 到 D 欄的資料，在記憶體中計算出 E 欄與 F 欄的結果，最後一次性寫回。

```
/**
 * 範例一：批次計算個股漲跌幅與量能狀態
 * 適用情境：台股盤後或盤中定時全面更新指標
 */
function batchCalculateStockData() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName("股市資料");
  
  // 取得該工作表最後一行與最後一欄的編號
  const lastRow = sheet.getLastRow();
  
  // 防止工作表完全沒有資料時引發錯誤 (過濾掉只有標題列的情況)
  if (lastRow <= 1) {
    Logger.log("沒有足夠的股票資料可供計算。");
    return;
  }
  
  // 【步驟 1：批次讀取】
  // 從第 2 行、第 1 欄開始，讀取到最後一行（扣除標題列），共 4 欄 (A 到 D)
  // 包含：股票名稱(A)、昨收(B)、現價(C)、成交量(D)
  const sourceRange = sheet.getRange(2, 1, lastRow - 1, 4);
  const sourceData = sourceRange.getValues();
  
  // 建立一個空的二維陣列，用來存放要寫回 E 欄與 F 欄的結果
  const outputData = [];
  
  // 設定一個簡單的量能異常門檻（例如成交量大於 5000 張視為放大）
  const VOLUME_THRESHOLD = 5000;
  
  // 【步驟 2：記憶體中進行陣列計算】
  for (let i = 0; i < sourceData.length; i++) {
    const row = sourceData[i];
    const stockName = row[0];
    const yesterdayClose = row[1];
    const currentPrice = row[2];
    const volume = row[3];
    
    let priceChangePercent = 0;
    let volumeStatus = "正常";
    
    // 安全機制：防止分母為 0 或資料缺失
    if (yesterdayClose && yesterdayClose > 0) {
      // 漲跌幅計算公式：((現價 - 昨收) / 昨收) * 100
      priceChangePercent = ((currentPrice - yesterdayClose) / yesterdayClose) * 100;
      // 四捨五入到小數點後兩位
      priceChangePercent = Math.round(priceChangePercent * 100) / 100;
    }
    
    // 量能狀態判定
    if (volume >= VOLUME_THRESHOLD) {
      volumeStatus = "量能爆發";
    } else if (volume < 500) {
      volumeStatus = "窒息量";
    }
    
    // 將計算結果包裝成一個內層陣列 [漲跌幅, 量能狀態]，對應 E 欄與 F 欄
    outputData.push([priceChangePercent, volumeStatus]);
  }
  
  // 【步驟 3：批次寫入】
  // 目標範圍：從第 2 行開始，第 5 欄 (E欄)，行數為 outputData 的長度，欄數為 2 (E, F欄)
  const targetRange = sheet.getRange(2, 5, outputData.length, 2);
  
  // 一次性寫回試算表
  targetRange.setValues(outputData);
  
  Logger.log(`成功批次處理 ${outputData.length} 檔個股資料！`);
}
```

> 範例二：進階 JS 陣列優化 ── 使用 map()

現代 V8 引擎後，老舊的 for 迴圈與手動 push 顯得有些冗長。我們可以使用 JavaScript 的 map() 方法，直接將「輸入二維陣列」轉換成「輸出二維陣列」，程式碼會變得極具可讀性且更不易出錯。

```
/**
 * 範例二：利用 Array.prototype.map() 進行函式庫風格的批次優化
 */
function batchCalculateWithMap() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("股市資料");
  const lastRow = sheet.getLastRow();
  if (lastRow <= 1) return;

  // 一口氣讀取整張表 A2:D (從第2行,第1欄開始，讀取 lastRow-1 行，共4欄)
  const data = sheet.getRange(2, 1, lastRow - 1, 4).getValues();
  
  // 使用 map 直接映射出結果陣列（精準、不易發生索引混淆）
  const results = data.map(row => {
    // 利用解構賦值直接抓出變數
    const [name, yesterdayClose, currentPrice, volume] = row;
    
    if (!yesterdayClose) return [0, "無資料"];
    
    const change = (((currentPrice - yesterdayClose) / yesterdayClose) * 100).toFixed(2);
    const status = volume > 10000 ? " 特大單湧入" : (volume > 5000 ? " 量能放大" : "常態");
    
    // 回傳的值會自動組裝成新的二維陣列元素
    return [parseFloat(change), status];
  });
  
  // 批次寫入 E2:F (第2行, 第5欄)
  sheet.getRange(2, 5, results.length, 2).setValues(results);
  Logger.log("Map 批次計算完成。");
}
```

> 範例三：逐行寫入 vs 批次寫入之時間觀測器

同時執行「逐行處理」與「批次處理」，並精確算出兩者消耗的時間差。你可以在擁有 100 筆以上資料的工作表上執行這個函式，透過日誌數據，你可觀察被批次處理的威力。

```
/**
 * 範例三：效能測試──對比逐行讀寫與批次處理的耗時差異
 */
function performanceShowdown() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("股市資料");
  const lastRow = sheet.getLastRow();
  if (lastRow <= 1) return;
  
  // ==========================================
  // 測試 A：傳統逐行讀寫 (效能殺手型寫法)
  // ==========================================
  const startSlow = Date.now();
  
  for (let r = 2; r <= lastRow; r++) {
    // 每次迴圈都在呼叫 API 讀取
    const yesterdayClose = sheet.getRange(r, 2).getValue();
    const currentPrice = sheet.getRange(r, 3).getValue();
    
    let change = 0;
    if (yesterdayClose) {
      change = ((currentPrice - yesterdayClose) / yesterdayClose) * 100;
    }
    
    // 每次迴圈都在呼叫 API 寫入
    sheet.getRange(r, 5).setValue(change); 
  }
  
  const endSlow = Date.now();
  const slowDuration = (endSlow - startSlow) / 1000; // 換算成秒
  
  // ==========================================
  // 測試 B：全陣列批次處理 (現代優化寫法)
  // ==========================================
  const startFast = Date.now();
  
  // 1 次讀取
  const rawData = sheet.getRange(2, 2, lastRow - 1, 2).getValues(); 
  
  // 記憶體中極速運算
  const fastResults = rawData.map(row => {
    const [yClose, cPrice] = row;
    const change = yClose ? ((cPrice - yClose) / yClose) * 100 : 0;
    return [change];
  });
  
  // 1 次寫入
  sheet.getRange(2, 5, fastResults.length, 1).setValues(fastResults);
  
  const endFast = Date.now();
  const fastDuration = (endFast - startFast) / 1000;
  
  // ==========================================
  // 印出結果報告
  // ==========================================
  Logger.log("=== 效能對決報告 ===");
  Logger.log(`資料總筆數：${lastRow - 1} 筆`);
  Logger.log(`逐行讀寫耗時：${slowDuration.toFixed(3)} 秒`);
  Logger.log(`批次處理耗時：${fastDuration.toFixed(3)} 秒`);
  Logger.log(`效能提升幅度：${(slowDuration / fastDuration).toFixed(1)} 倍！`);
}
```

##### 執行步驟
###### 第一步：開啟 Apps Script 編輯器從剛才建立好資料的 Google 試算表直接點進去：
1. 開啟你剛剛貼好股市資料的 Google 試算表。
2. 點選頂部功能表的 「擴充功能」 (Extensions) -> 「Apps Script」。
3. 瀏覽器會自動彈出一個全新的分頁，這就是你的 GAS 雲端開發環境。

###### 第二步：貼上程式碼
1. 進入編輯器後，你會在左側看到一個名為 程式碼.gs 的檔案，中間的程式碼視窗預設會有一段：
```
function myFunction() {

}
```
2. 把中間視窗內原本的 myFunction() 全部刪除（清空）。
3. 複製範例一提供的那整段 batchCalculateStockData() 程式碼，完整貼入這個空白的視窗中。
4. 貼上後，按下鍵盤快速鍵 Ctrl + S (Windows) 或 Cmd + S (Mac) 進行專案儲存（或者點選畫面上方像「小磁碟片」的儲存圖示）。
- 注意：只要檔案名稱旁邊出現「橘色小圓點」，就代表程式碼有更動但尚未儲存。

###### 第三步：如何編譯與運行
在 GAS 中，JavaScript 是即時直譯執行的，不需像 Java 或 C++ 那樣手動進行繁瑣的編譯（語法檢查會在儲存與執行時由後台 V8 引擎自動完成）。

1. 選擇你要執行的函式在網頁頂部的控制列，你會看到一個下拉選單（旁邊寫著「偵錯」與「執行」）。請點開它，確認選中的是 batchCalculateStockData。
2. 點選「執行」
   點選下拉選單左側的 「 執行」(Run) 按鈕（外觀是一個向右的箭頭 ▶）。
3. 【關鍵】首次執行的「授權驗證」（僅需做一次）
   因為這段程式碼會去讀取和修改你的 Google 試算表，Google 為了安全，在第一次運行時會跳出安全性警告。請依照以下步驟點選：
   - 畫面跳出「需要授權」視窗 $\rightarrow$ 點選 「審閱權限」 (Review Permissions)。
   - 選擇你目前登入的 Google 帳號。
   - 畫面會出現一個看起來很可怕的警告「Google 尚未驗證此應用程式」 $\rightarrow$ 請點選左下角小小的 「進階」 (Advanced)。
   - 在下方展開的文字中，點選 「前往『未命名專案』(不安全)」 (Go to Untitled project (unsafe))。
   - 最後，點選 「允許」 (Allow)。

###### 第四步：查看結果
授權完成後，程式就會正式啟動。
- 看日誌： 編輯器下方會自動彈出「執行日誌」視窗。當你看到出現 成功批次處理 10 檔個股資料！ 以及 執行已完成 的字樣，就代表程式完美運行成功。
- 看試算表： 這時切換回你的 Google 試算表分頁，你會發現原本空白的 E 欄（漲跌幅） 與 F 欄（量能狀態） 已經在一瞬間被填上了正確的數據！

## UI 客製化與自動化選單
### 簡單觸發器 (Simple Triggers)
在 GAS 中，觸發器（Triggers）就像是監聽器，當特定事件發生時，Google 的雲端伺服器會自動呼叫對應的函式。GAS 的觸發器分為兩大類：簡單觸發器 (Simple Triggers) 與 可安裝觸發器 (Installable Triggers)。
onOpen(e) 屬於簡單觸發器。它的特點是：
- 名稱綁定： 只要將函式名稱精準命名為 onOpen(e)，不需做任何額外設定，Google 就會在使用者打開該試算表時自動執行它。
- 參數 e 的含意： e 代表事件物件（Event Object），裡面包含了觸發當時的上下文資訊（例如是哪位使用者、哪個試算表被打開）。在初階應用中，我們通常不需要深入解析 e，但保留這個參數是良好的開發習慣。
- 權限限制 (Auth Modes)： 簡單觸發器運行在受限的 AuthMode.LIMITED 模式下。這意味著它不能存取需要個人隱私權限的服務（例如發送 Gmail、存取個人雲端硬碟），但它完全有權限在當前的試算表中建立自訂選單與彈出視窗。這項安全機制保護了使用者，也讓自訂選單的部署變得極其安全。

#### 試算表 UI 類別 (SpreadsheetApp.getUi)
要改造 Google 試算表的外觀，我們必須透過 SpreadsheetApp.getUi() 取得當前工作環境的用戶介面物件。這個 Ui 物件是 GAS 提供給開發者的畫布，透過它，我們可以實現以下幾種客製化互動：
1. 自訂選單 (Menus)： 在試算表頂部（與「檔案」、「編輯」、「擴充功能」並列）新增自訂的選單項目。
2. 快顯回應視窗 (Alerts / Prompt Dialogs)： 彈出對話框，向使用者確認執行意圖（例如：「確定要清除今日資料嗎？」）或讓使用者輸入參數（例如：「請輸入要查詢的股票代號」）。
3. 側邊欄 (Sidebars) 與 網頁對話框 (Modal Dialogs)： 透過撰寫 HTML/CSS，在右側或中央拉出極度複雜的互動圖表（如使用 D3.js 繪製 K 線圖），這是高階 GAS 開發的核心。
我們聚焦在實用、建置成本最低的自訂選單 (Menus)。

#### 自訂選單的鏈式語法 (Fluent / Chaining API)
GAS 在建構 UI 時採用了鏈式程式設計 (Fluent API)，這讓代碼看起來層次分明。我們會用到以下三個核心方法：
- createMenu(caption)：在頂部建立一個大分類選單，caption 就是顯示在功能列上的主名稱。
- addItem(caption, functionName)：在大分類下新增一個按鈕。caption 是按鈕名稱，functionName 則是當使用者點擊這個按鈕時，背後要呼叫的 GAS 函式名稱字串（注意：這裡必須用引號包裹，不能加括號）。
- addSeparator()：在按鈕與按鈕之間畫一條灰色的橫向分隔線，用於功能分類。
- addToUi()：它是整段鏈式語法的終點。如果沒呼叫 addToUi()，前面建構的所有選單都只會存在於伺服器的記憶體中，而不會真正渲染到使用者的試算表畫面上。

### 自動化選單建構與異常處理機制

#### 選單的生命週期與動態渲染
理解選單的生命週期，能幫你省去未來大量的偵錯時間：

1. 使用者載入網頁： 當瀏覽器打開試算表時，Google 伺服器在後台啟動 V8 引擎，並自動尋找程式碼中是否有 onOpen()。
2. 執行掛載： 執行 onOpen() 內的 addToUi()，此時瀏覽器的功能列上會刷出你的專屬股市工具選單。
3. 記憶體留存： 只要這個網頁分頁沒有被重新整理或關閉，這個選單就會一直卡在功能列上。
4. 點擊觸發： 當交易員點選「計算今日指標」時，GAS 才會真正去執行對應的運算函式。

> 開發時的常見問題：為什麼我寫好選單了，畫面上卻沒出現？

因為 onOpen 只有在「打開活頁簿」的那一刻會觸發。當你在 Apps Script 編輯器裡剛寫完程式碼並儲存時，你的試算表網頁早就已經打開了，所以它不會平白無故冒出來。
解法： 你不需要反覆重新整理試算表網頁。在編輯器中，你可以直接在頂部下拉選單選擇 onOpen 並手動點擊「▶ 執行」，這能強迫它在當前畫面渲染出選單，是極有效率的開發小技巧。

#### 介面優化：加入彈出式確認視窗 (Ui.alert)
當一個自動化腳本威力強大到會「一鍵覆蓋整張表的資料」時，良好的 UI 設計絕對不能讓使用者輕易誤觸。我們必須在執行運算前，加入一道防禦性設計——安全確認對話框。

透過 ui.alert(title, prompt, buttons)，我們可以暫停程式執行，彈出一個視窗詢問使用者。

```
const ui = SpreadsheetApp.getUi();
const response = ui.alert("警告", "這將會覆蓋現有的分析數據，確定執行嗎？", ui.ButtonSet.YES_NO);

if (response == ui.Button.NO) {
  // 使用者退出了，終止程式
  return; 
}
```
這種設計能大幅降低操作人員誤觸、導致辛辛苦苦保存的歷史股市資料被洗掉的悲劇。

### 程式範例說明
請將以下程式碼全面覆蓋或接續貼在 Apps Script 編輯器中。這裡我們將「範例一的計算邏輯」、「一個清除資料的實用工具」，以及「安全提示視窗」完美整合進自訂選單中。

```
/**
 * ============================================================================
 * 建立試算表專屬的股市分析工具列
 * ============================================================================
 */

/**
 * 簡單觸發器：當試算表被開啟時，自動執行此函式掛載 UI 選單
 * @param {Object} e 事件物件
 */
function onOpen(e) {
  // 取得當前試算表的用戶介面物件
  const ui = SpreadsheetApp.getUi();
  
  // 開始建構自訂選單
  ui.createMenu("股市自動化工具")
    .addItem("批次計算個股指標", "uiTriggerCalculate")
    .addSeparator() // 畫一條美麗的分隔線
    .addItem("清除分析結果 (E, F欄)", "uiTriggerClearData")
    .addToUi(); // 核心：將選單真正刷入試算表介面
}

/**
 * UI 按鈕專用：執行股市指標計算 (包裝了安全確認機制)
 */
function uiTriggerCalculate() {
  const ui = SpreadsheetApp.getUi();
  
  // 1. 跳出防誤觸確認視窗
  const result = ui.alert(
    "執行確認", 
    "您即將針對【股市資料】工作表進行全面的批次漲跌幅與量能判定，是否繼續？", 
    ui.ButtonSet.YES_NO
  );
  
  // 2. 如果使用者點選「否(NO)」或是直接關閉視窗，則終止程式
  if (result === ui.Button.NO || result === ui.Button.CLOSE) {
    // 透過吐司點心訊息 (Toast) 提示使用者取消操作，顯示 3 秒
    SpreadsheetApp.getActiveSpreadsheet().toast("操作已取消", "系統通知", 3);
    return;
  }
  
  // 3. 使用者確認要執行，呼叫我們先前寫好的核心批次計算函式
  try {
    // 這裡調用上一單元精進後的 Map 版本，效能最好
    executeCoreCalculation(); 
    
    // 4. 成功後跳出提示
    ui.alert("計算完成", "所有個股的漲跌幅與量能狀態已順利寫入！", ui.ButtonSet.OK);
  } catch (error) {
    ui.alert("發生錯誤", `計算過程中出現非預期錯誤：${error.toString()}`, ui.ButtonSet.OK);
  }
}

/**
 * UI 按鈕專用：清除工作表中的 E 欄與 F 欄，方便使用者重新測試
 */
function uiTriggerClearData() {
  const ui = SpreadsheetApp.getUi();
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName("股市資料");
  
  const lastRow = sheet.getLastRow();
  if (lastRow <= 1) {
    ss.toast("目前工作表沒有資料需要清除。", "系統通知");
    return;
  }
  
  // 強烈警告視窗
  const result = ui.alert(
    "危險操作警告", 
    "此操作將會「徹底清空」工作表中所有的漲跌幅與量能判定數據，此動作無法完全還原！您真的確定嗎？", 
    ui.ButtonSet.YES_NO
  );
  
  if (result === ui.Button.YES) {
    // 清除從第 2 行開始的 E 欄與 F 欄 (第5欄開始，共計2欄)
    sheet.getRange(2, 5, lastRow - 1, 2).clearContent();
    ss.toast("E 欄與 F 欄資料已成功清空！", "系統通知", 3);
  } else {
    ss.toast("安全取消，未刪除任何資料。", "系統通知", 3);
  }
}

/**
 * 核心計算邏輯 (分離架構：UI 歸 UI，邏輯歸邏輯)
 * 改造自單元一的優良批次處理代碼
 */
function executeCoreCalculation() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("股市資料");
  const lastRow = sheet.getLastRow();
  if (lastRow <= 1) return;

  // 1 次讀取 A2:D 欄
  const data = sheet.getRange(2, 1, lastRow - 1, 4).getValues();
  
  // 記憶體極速運算
  const results = data.map(row => {
    const [name, yesterdayClose, currentPrice, volume] = row;
    if (!yesterdayClose || yesterdayClose <= 0) return [0, "無資料"];
    
    const change = (((currentPrice - yesterdayClose) / yesterdayClose) * 100).toFixed(2);
    
    let status = "正常";
    if (volume >= 5000) {
      status = "量能爆發";
    } else if (volume < 500) {
      status = "窒息量";
    }
    
    return [parseFloat(change), status];
  });
  
  // 1 次寫入 E2:F 欄
  sheet.getRange(2, 5, results.length, 2).setValues(results);
}
```
### 程式範例詳細拆解與運行說明
1. 職責分離的架構 (Separation of Concerns)
   在此範例中，請特別注意到不要把計算公式硬塞在 onOpen 或 UI 觸發函式裡。我們建立了一個獨立的 executeCoreCalculation()。
    - uiTriggerCalculate() 只負責跟使用者聊天（跳出 Alert、確認按鈕、顯示成功視窗）。
    - executeCoreCalculation() 只負責跟數據打交道（讀取、Map、寫回）。這種架構的好處在於，未來如果你想把這套股市計算邏輯改成「每天下午兩點半自動定時執行」，你只需要直接將觸發器綁定到 executeCoreCalculation() 即可，完全不需要去修改任何 UI 代碼，這是專業開發者必須具備的專案規劃能力。

2. 實機運行與驗證步驟
為了在 Google 試算表上看見這套工具列，請按照以下流程操作：
    1. 儲存代碼： 在 Apps Script 編輯器貼上上述代碼後，點選「儲存」圖示。
    2. 模擬開表觸發： 在頂部函式選單中選擇 onOpen，然後點擊 「▶ 執行」。這會模擬使用者重新打開這張表的動作。
    3. 切換回你的 Google 試算表網頁。此時，在頂部系統選單的最後方（通常在「說明」右側），會突然神奇地蹦出一個名為 股市自動化工具 的全新選單！

3. 使用者測試流程
現在你可以扮演一位真正的交易員，測試以下兩種情境：</br>
    - 情境 A（測試防誤觸與計算）：
      1. 點選 股市自動化工具 $\rightarrow$ 點擊 批次計算個股指標。
      2. 畫面上會跳出 Google 原生對話框，詢問你是否繼續。

## 進階事件驅動與即時警示

### onEdit(e)

在 Google Apps Script 中，onEdit(e) 是一個簡單觸發器（Simple Trigger）。當使用者在試算表中「手動修改任意儲存格內容」時，Google 伺服器會自動偵測到這個動作，並主動呼叫 onEdit 函式。

最重要的是，Google 會順便打包一個名為 e 的「事件物件」傳遞給程式，裡面包含了所有被修改的關鍵資訊：
- e.range：被修改的儲存格範圍（Range 物件），是我們調整顏色、取得座標的最重要工具。
- e.value：修改後的「新數值」。
- e.oldValue：修改前的「舊數值」（若原本是空的則為 undefined）。
- e.source：當前的試算表物件（Spreadsheet 物件）。

> 關鍵限制與陷阱
- 自動公式不觸發：onEdit 只能捕捉「人為手動輸入」或「透過 API 蓋寫固定值」的動作。如果儲存格內是 =GOOGLEFINANCE() 或 =A1+B1 等公式，當公式因為時間或連動而改變數值時，是不會觸發 onEdit 的！
- 權限限制：簡單觸發器運行在受限模式下，無法執行需要使用者授權的操作（例如寄送 Gmail、存取外部 API）。若後續需要連動寄信，必須改用「可安裝的觸發器（Installable Trigger）」。

### 課程範例
開啟試算表的擴充功能「Apps Script」，逐步加入以下三個範例。

> 範例一：單一欄位防守與即時漲跌幅視覺標記情境需求：

在 ch08 試算表中，欄位 C 為「目前現價」，欄位 B 為「昨收價」 。當我們手動手動更新或透過 API 自動覆寫「目前現價」（第 3 欄）時，程式需要自動去比對該列的「昨收價」（第 2 欄），並依照漲跌幅度將現價儲存格改變顏色（漲：紅底白字；跌：綠底白字；平盤：恢復原本樣式） 。  

```
/**
 * 範例一：監控「目前現價」欄位並實時切換紅綠燈視覺警示
 * 修正點：明確處理平盤時的字體顏色、粗細回復，避免 null 失效
 */
function monitorStockPriceVisuals(e) {
  if (!e) return;

  var range = e.range;
  var sheet = range.getSheet();
  if (sheet.getName() !== "股市資料") return;

  var row = range.getRow();
  var column = range.getColumn();

  // 鎖定第 3 欄（目前現價）
  if (row > 1 && column === 3) {
    var yesterdayPrice = parseFloat(sheet.getRange(row, 2).getValue());
    var currentPrice = parseFloat(e.value); // 修改後的新現價

    // 如果被清空或不合法，直接全面重設
    if (isNaN(currentPrice) || isNaN(yesterdayPrice) || yesterdayPrice === 0) {
      resetCColumnStyle(range);
      return;
    }

    var changePercent = (currentPrice - yesterdayPrice) / yesterdayPrice;

    if (changePercent > 0) {
      // 漲：紅底、白字、粗體
      range.setBackground("#FF4D4D")
           .setFontColor("#FFFFFF")
           .setFontWeight("bold");
    } else if (changePercent < 0) {
      // 跌：綠底、白字、粗體
      range.setBackground("#2ECC71")
           .setFontColor("#FFFFFF")
           .setFontWeight("bold");
    } else {
      // 平盤：呼叫專屬重設函式，強制黑字、常規、無背景
      resetCColumnStyle(range);
    }
  }
}
```
> 範例二：整列觸發視覺警示（全列變色追蹤）

情境需求：單一儲存格變色有時候不夠醒目。我們希望當某檔股票的「目前現價」發生變動，且漲幅或跌幅超過 3% 臨界點時，將「整列欄位（A到D）」都塗上淡紅色或淡綠色背景，讓使用者在一整張大型交易看板中，能瞬間鎖定產生劇烈波動的標的。

```
/**
 * 範例二：當波動超過 3% 時，進行周邊欄位（A, B, D）動態標記
 * 修正點：改為分別著色 A, B 欄與 D 欄，故意避開 C 欄（第3欄），防止背景顏色衝突！
 */
function monitorRowAlertThreshold(e) {
  if (!e) return;

  var range = e.range;
  var sheet = range.getSheet();
  if (sheet.getName() !== "股市資料") return;

  var row = range.getRow();
  var column = range.getColumn();

  if (row > 1 && column === 3) {
    var currentPrice = parseFloat(e.value);
    var yesterdayPrice = parseFloat(sheet.getRange(row, 2).getValue());

    if (isNaN(currentPrice) || isNaN(yesterdayPrice) || yesterdayPrice === 0) {
      // 異常時，把周邊欄位也重設為白底
      sheet.getRange(row, 1, 1, 2).setBackground("#FFFFFF");
      sheet.getRange(row, 4).setBackground("#FFFFFF");
      return;
    }

    var changePercent = (currentPrice - yesterdayPrice) / yesterdayPrice;
    var threshold = 0.03; // 3% 門檻

    // 【核心改造】取得周邊範圍，避開第 3 欄 (C欄)
    var leftSideRange = sheet.getRange(row, 1, 1, 2); // A 到 B 欄
    var rightSideRange = sheet.getRange(row, 4);      // D 欄

    if (changePercent >= threshold) {
      // 劇烈上漲：周邊塗淡紅色
      leftSideRange.setBackground("#FFCCCC");
      rightSideRange.setBackground("#FFCCCC");
    } else if (changePercent <= -threshold) {
      // 劇烈下跌：周邊塗淡綠色
      leftSideRange.setBackground("#E2F7E3");
      rightSideRange.setBackground("#E2F7E3");
    } else {
      // 波動內：周邊恢復常態白底
      leftSideRange.setBackground("#FFFFFF");
      rightSideRange.setBackground("#FFFFFF");
    }
  }
}

function resetCColumnStyle(range) {
  range.setBackground(null)          // 移除背景
       .setFontColor("#000000")      // 強制字體變回黑色
       .setFontWeight("normal");     // 強制變回普通粗細
}
```

> 範例三：成交量爆量即時警示與防錯回復情境需求：

除了價格，成交量往往是策略突破的關鍵。當更動第 4 欄「今日成交量」時 ，若發現該個股今日成交量大於 20,000 張（在我們範例中代表爆量突破標的） ，自動在該量能儲存格加上亮黃色背景。
同時，為了防止人為輸入負數或異常錯誤字串，增加防錯自動回復機制：若輸入小於 0 的數值，自動將數據強制回復為「修改前的值（e.oldValue）」並彈出警告。

```
/**
 * 範例三：成交量爆量亮黃燈警告與負數輸入防錯回復
 */
function monitorVolumeAlert(e) {
  if (!e) return;

  var range = e.range;
  var sheet = range.getSheet();
  if (sheet.getName() !== "股市資料") return;

  var row = range.getRow();
  var column = range.getColumn();

  if (row > 1 && column === 4) {
    var inputVolume = parseFloat(e.value);

    if (inputVolume < 0) {
      SpreadsheetApp.getUi().alert("錯誤：成交量不能為負數！資料已自動復原。");
      range.setValue(e.oldValue);
      return;
    }

    if (inputVolume >= 20000) {
      range.setBackground("#FFFF99").setFontColor("#000000");
    } else {
      range.setBackground(null).setFontColor("#000000");
    }
  }
}
```
在同一個 Google Apps Script 專案中，不能存在兩個叫做 function onEdit(e) 的函式，否則後面的會直接覆蓋掉前面的，導致功能失效。

為了保持原本 ch08 功能的完整性，我們採用主控分配模式（Dispatcher Pattern）。我們在專案中只建立一個主 onEdit(e)，並在裡面依序呼叫我們剛才寫好的三個獨立功能。這樣做不僅結構清晰，未來要關閉或新增其中一個策略，只要在主程式註解掉即可。

```
/**
 * Google 試算表內建簡單觸發器主入口
 * 此架構會自動分流執行各個模組，完全不影響原先試算表既有的其他獨立函式功能
 */
function onEdit(e) {
  // 執行範例一：現價即時紅綠視覺標記
  monitorStockPriceVisuals(e);
  
  // 執行範例二：超過 3% 波動之全列大範圍變色警示
  monitorRowAlertThreshold(e);
  
  // 執行範例三：成交量爆量警示與輸入防錯回復
  monitorVolumeAlert(e);

  // ==========================================
  // 您原先若有其他需要在 onEdit 時觸發的程式，
  // 可以直接寫在下方，彼此完全不相干擾。
  // ==========================================
}
```
