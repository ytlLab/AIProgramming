# 進階自動化通知系統

## Discord Bot建立與通訊
- Webhook： 是一種「單向、被動」的管道。它提供一個專屬的 URL，任何人只要向這個網址發送 POST 請求，訊息就會丟進頻道。優點是免驗證、開發極快；缺點是功能單一，無法處理互動（如按鈕、選單）或讀取頻道狀態。
- Bot Account： 是一個「虛擬的 Discord 用戶」。它擁有獨立的 Token（金鑰）作為身分識別，透過 Discord 官方的 REST API 與 Gateway（WebSocket）進行雙向通訊。Bot 不僅能發送高度客製化的結構化訊息（Embeds），未來還能監聽使用者的關鍵字指令、管理頻道成員或建立斜線指令（Slash Commands）。

### 建立 Discord 測試用伺服器
1. 開啟 [Discord](https://discord.com/) 應用程式或網頁版
2. 建立新伺服器：在左側伺服器列表的最下方，點擊 「+」（新增伺服器） 按鈕。
    - 選擇 「親自建立」。
    - 選擇 「我和我的朋友」。
    - 輸入伺服器名稱（例如：testBot），然後點擊 「建立」。

### 建立 Discord Application 與 Bot
1. 瀏覽器開啟 [Discord Developer Portal](https://discord.com/developers/home) 並登入帳號。
2. 點擊右上角的 New Application 鈕。
3. 在彈出視窗中輸入應用程式名稱（例如：ytlAi_Stock_Alert），勾選同意條款後點擊 Create。
4. 進入專案後，點選左側選單的 Bot 標籤頁。
5. 找到 Token 區塊，點擊 Reset Token。系統會生成一串長字串（如 MTkyOD...），這是這台機器人的唯一身分憑證。請立刻複製並妥善保存，一旦離開此畫面，基於安全機制將無法再次查看，只能重新生成。
6. 關鍵隱私設定（Intents）： 在同一個頁面下滑找到 Privileged Gateway Intents 區塊。請務必將 MESSAGE CONTENT INTENT 開啟（切換為藍色）。若未開啟此功能，機器人將無法正確讀取或發送特定的複雜訊息內容。

### 權限配置與邀請機器人入群
Discord 採用 OAuth 2.0 安全架構。要將機器人加入伺服器，必須透過特定 URL 進行授權：
1. 點選左側選單的 OAuth2 -> URL Generator。
2. 在 Scopes（範疇） 區塊中，勾選 bot（這代表我們要邀請的是一個機器人個體）。
3. 當勾選 bot 後，下方會彈出 Bot Permissions（機器人權限） 面板。為了符合資安的「最小權限原則（Principle of Least Privilege）」，我們不勾選 Administrator，而是精準勾選以下權限：
    - Send Messages（發送訊息）
    - Embed Links（嵌入鏈結，這對於發送精美字卡至關重要）
    - Read Message History（讀取歷史訊息）
4. 複製最下方 GENERATED URL 區塊中產生的長網址。
5. 開啟瀏覽器新分頁並貼上該網址，選擇你有管理員權限的 Discord 伺服器，點擊「認證」，完成人類驗證後，你就會看到機器人以「離線」狀態加入你的伺服器頻道中。

### Python 串接 Discord API
1. 底層通訊邏輯：</br>
不使用第三方封裝庫（如 discord.py，因為該庫採用非同步監聽架構，對於單純發通知的 Colab 腳本過於臃腫），而是直接使用底層的 HTTP REST API。</br>
使用 requests.post() 向 Discord 官方的 v10 版本端點發送請求。</br>
在 HTTP 請求中，必須加入特殊的 Headers 結構：
    - Authorization: 其值必須為 "Bot <你的Token>"（注意 Bot 與 Token 之間有一個半形空格）。
    - Content-Type: 必須為 "application/json"，告訴 Discord 伺服器我們傳送的是 JSON 格式的結構化資料。

2. 進階資料結構：Embed（嵌入字卡）
Discord 允許在訊息中加入一個名為 embeds 的陣列（Array），裡面包含一個或多個字典（Dictionary）。其核心欄位包括：
- title / description: 字卡的主標題與副標題。
- color: 側邊條的顏色，必須是 十進位整數 (Decimal Integer)。例如純紅色在十六進位是 #D9381E，轉換為十進位就是 14235678。
- fields: 一個陣列，每個物件代表一個資料格。包含 name（小標題）、value（內容）、inline（布林值，決定是否與相鄰欄位排在同一行）。
- timestamp: 符合 ISO 8601 格式的國際標準時間字串（例如 2026-06-05T06:00:00Z）。

```
import requests
import json
import datetime

def send_advanced_discord_alert(stock_id, price, change, volume, reason):
    """
    透過 Discord REST API 發送高度客製化的股市即時警報字卡
    """
    # ---------------------------------------------------------
    # 配置區 (請替換為你的真實資料)
    # ---------------------------------------------------------
    BOT_TOKEN = "YOUR_DISCORD_BOT_TOKEN_HERE"
    CHANNEL_ID = "YOUR_DISCORD_CHANNEL_ID_HERE"
    
    # Discord API v10 傳送訊息端點
    api_url = f"https://discord.com/api/v10/channels/{CHANNEL_ID}/messages"
    
    # 建構安全憑證與通訊協定標頭
    headers = {
        "Authorization": f"Bot {BOT_TOKEN}",
        "Content-Type": "application/json"
    }
    
    # ---------------------------------------------------------
    # 動態視覺邏輯：根據漲跌幅切換字卡邊框顏色
    # ---------------------------------------------------------
    # 漲 (正數) -> 紅色 (十進位: 14177041); 跌 (負數) -> 綠色 (十進位: 3066993)
    if change >= 0:
        embed_color = 14177041  # Crimson Red
        status_icon = "📈"
    else:
        embed_color = 3066993   # Emerald Green
        status_icon = "📉"
        
    # 獲取當前 UTC 時間，並格式化為 ISO 8601 標準格式
    current_utc_time = datetime.datetime.utcnow().isoformat() + "Z"
    
    # ---------------------------------------------------------
    # 構建符合 Discord API 規範的複雜 JSON Payload
    # ---------------------------------------------------------
    payload = {
        "content": f"⚡ **【ytlAi Lab 智能監控中心】** 偵測到行情異動！",
        "embeds": [
            {
                "title": f"{status_icon} 行情觸發警報：{stock_id}",
                "description": "系統已達到您預設的技術指標策略條件，詳細數據如下：",
                "color": embed_color,
                "fields": [
                    {
                        "name": "💰 當前成交價",
                        "value": f"**${price:,.2f} TWD**",
                        "inline": True
                    },
                    {
                        "name": "📊 今日漲跌幅",
                        "value": f"`{'+' if change >= 0 else ''}{change}%`",
                        "inline": True
                    },
                    {
                        "name": "📦 當日成交量",
                        "value": f"{volume:,} 張",
                        "inline": True
                    },
                    {
                        "name": "🔍 策略觸發原因",
                        "value": f"```fix\n{reason}\n```",
                        "inline": False
                    }
                ],
                "footer": {
                    "text": "ytlAi Lab 訊號發送引擎 v2.1"
                },
                "timestamp": current_utc_time
            }
        ]
    }
    
    # 執行 POST 請求傳送資料
    try:
        response = requests.post(api_url, headers=headers, data=json.dumps(payload))
        
        # 檢查 HTTP 狀態碼
        if response.status_code == 200:
            print("🎉 [成功] Discord 進階警報字卡已成功送達指定頻道！")
        elif response.status_code == 401:
            print("❌ [錯誤 401] 身分驗證失敗！請檢查你的 BOT_TOKEN 是否填寫正確。")
        elif response.status_code == 403:
            print("❌ [錯誤 403] 權限不足！請確認機器人是否已被成功邀請入該伺服器，且擁有發送訊息的權限。")
        elif response.status_code == 404:
            print("❌ [錯誤 404] 找不到目標！請檢查 CHANNEL_ID 是否正確無誤。")
        else:
            print(f"❌ [未知錯誤] 狀態碼: {response.status_code}, 回應內容: {response.text}")
            
    except Exception as e:
        print(f"💥 連線發生異常: {e}")

# ---------------------------------------------------------
# 主程式測試測試執行
# ---------------------------------------------------------
if __name__ == "__main__":
    print("開始測試進階發送功能...")
    # 模擬台積電觸發個案
    send_advanced_discord_alert(
        stock_id="2330 台積電",
        price=924.00,
        change=2.67,
        volume=48215,
        reason="K值(82)大於D值(75)呈黃金交叉，且成交量較昨日放大1.5倍。"
    )
```

```
def get_weekly_weather(county: str, district: str):
    """
    [Ch05 核心]：精準過濾遍歷版 - 100% 解決要素錯位問題
    """

        time_slots = target_element.get("Time", [])
        if not time_slots:
            return f"❌ 找不到【{county}{district}】的預報時間段資料。"

            element_values = slot.get("ElementValue", [{}])

            if not start_time:
                continue

            try:
```

## OAuth 2.0 觀念
在 Web 技術發展的早期，許多自動化程式確實是要求使用者提供明文的帳號與密碼。然而，這種做法在現代網路安全中被視為重大的災難。為了徹底解決這個安全痛點，OAuth 2.0（開放授權，Open Authorization 2.0） 應運而生。

### 密碼驗證（Password Authentication）風險
- 密碼過度暴露（Password Exposure）：
    - 第三方應用程式必須以明文（Plain Text）或可解密的形式，將使用者的密碼儲存在本地端檔案或雲端伺服器中。一旦這個程式碼不小心被上傳到公開的 GitHub 儲存庫，使用者的主要帳號密碼就會瞬間外洩。

- 權限無法限縮（Monolithic Access）：
    - 密碼代表的是使用者在該平台上的最高、最完整權限。當你把 Discord 密碼交給 Python 程式時，這個程式不僅可以替你「發送訊息」，還可以替你「刪除伺服器」、「修改個人密碼」、「讀取私密刷卡紀錄」等。程式擁有了它根本不需要的過大權限。

- 無法輕易撤銷（Difficult to Revoke）：
    - 如果你發現某個 Python 自動化腳本可能被植入了惡意程式，你想要阻止它繼續存取你的帳號，你唯一的辦法就是「更改你的 Discord 主帳號密碼」。這會導致你所有其他合法的裝置（手機、iPad、電腦）全部被迫登出，造成極大的不便。

- 缺乏時間限制（No Expiration Mechanism）：
    - 密碼除非被使用者主動修改，否則它是永久有效的。這意味著自動化程式可以無限期地、隨時隨地存取你的資源，大幅增加了被駭客攔截並長期潛伏的風險。

### OAuth 2.0 概念：以「房卡（Token）」取代「帳號密碼」
為了克服密碼驗證的缺陷，OAuth 2.0 核心思想就是「解耦（Decoupling）身分驗證與權限授權」。

用「飯店入住」生活例子來對比這兩種機制的差異：
|  | 傳統密碼驗證 | OAuth 2.0 驗證機制 |
| :--- | :--- | :--- |
| 比較維度 | 交出身分證/印章 | 飯店房卡 |
| 比喻情境 | 把身分證與護照直接交給導遊，讓他隨時幫你辦理各種手續。  | 你在飯店櫃檯出示身分證，櫃檯確認身分後發給你一張房間房卡。  |
| 權限範圍 | 無限權限：拿到你身分證的人可以去銀行開戶、甚至賣掉你的財產。 | 限縮權限：房卡只能開「305 號房」的門、搭乘電梯，它無法用來去櫃檯提領飯店的現金。 |
| 權限撤銷 | 代價極高：如果導遊不可信，你必須去戶政事務所掛失並重辦身分證。 | 輕易撤銷：如果不小心把房卡弄丟，只需通知櫃檯將該房卡「報廢」，其餘身分不受影響。 |
| 時間限制 | 永久有效：直到你主動重辦身分證為止。| 自動過期：房卡在隔天中午 12:00 退房後，便會自動失效，無法再開門。 |
